# PL-002c: 编辑器集成（异步解析、快照与 patch）

> **状态：历史探索草案。** 当前仓库不按本文执行；若与现状冲突，请以当前代码、`README.md`、`docs\design\architecture.md` 以及 `.ai\` 目录中的现行计划为准。

## 目标
- 实现编辑器与解析/渲染管线的端到端集成，保证输入路径低延迟并能在后台异步解析并应用最小更新到渲染层。

## 设计要点
- 编辑器工作流程：
  1. 编辑控件（主线程）维护 working buffer。
  2. 在编辑发生后（按 debounce 或关键事件）创建 snapshot 并发送到解析后台任务（带 version id）。
  3. 后台任务解析并生成 DocumentModel；完成后对比当前版本，若仍有效则生成 patch 并通过主线程应用。
  4. 若后台版本过时，则丢弃解析结果（取消 token 或比较版本号）。

- 快照语义：快照为纯字符串与版本号（可包含光标偏移用于定位）。
- Patch 应为最小化更新描述，便于在 WebView 或原生渲染器中局部更新。

## 线程与取消策略
- 后台解析应可中途取消；使用解析任务 ID 或 cancellation token。主线程每次发起解析都增加版本号。
- 后台解析完成前若有新快照到达，则取消旧任务并以新快照重建任务。

## 编辑器内交互要点
- 在编辑过程中尽量不阻塞主线程；语法高亮可做本地轻量化实现，复杂渲染通过 preview 完成。
- 提供滚动同步：当编辑器光标移到新的可见区域时，可触发 preview 滚动（带小延迟或交互锁）。

## 任务拆解与估时
- T1: 编辑器 snapshot 接口与版本管理（0.5d）
- T2: 实现 parseAsync 调度与 cancel（0.5d）
- T3: 实现 patch 应用层（针对 HTML Preview 与原生 Renderer 两种路径）（1d）
- T4: 滚动同步与光标定位（0.5d）
- T5: 集成端到端 demo（编辑 → 异步解析 → preview 更新）（0.5d）

估时合计：约 3 天

## 验收标准
- 快速键入（持续输入 > 1s）期间 UI 无明显卡顿，后台解析能稳定完成并更新 preview（过期结果被丢弃）。
- Patch 在 Preview 中表现为最小更新（不频繁重绘整页）。
