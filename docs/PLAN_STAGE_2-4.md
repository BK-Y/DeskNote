### 阶段 4：文件读写 + 失焦自动保存 (Day 11-12) ⏳ 排期中

**目标**：便签内容能存盘，下次启动恢复。每次失去焦点时自动保存。

**文件命名规则**：`dn_YYYYMMDD_###.md`

- `dn` = DeskNote 前缀
- `YYYYMMDD` = 创建日期（如 20250120）
- `###` = 当日序号（001, 002...），首次创建从 001 开始

示例：
- `dn_20250120_001.md` — 1月20日第一个便签
- `dn_20250120_002.md` — 1月20日第二个便签
- `dn_20250201_001.md` — 2月1日第一个便签

启动时读取 settings.json 中的 `last_file` 恢复上次编辑的文件。

| 步骤 | 内容 | 学习点 |
|------|------|--------|
| 4.1 | 从 settings.json 读取 `last_file` 键 | 读配置，没有则新建 `note.md` |
| 4.2 | 文件名格式 `dn_YYYYMMDD_###.md`，无文件时按日期+序号创建 | `GetDateFormatW` 获取日期 |
| 4.3 | 读取 .md 内容 → 显示到编辑器 | `ReadFile` / `SetWindowTextW` |
| 4.4 | 主窗口失去焦点时自动保存 | `WM_ACTIVATE` 检测 `WA_INACTIVE` |
| 4.5 | 获取编辑器内容写入 .md 文件 | `GetWindowTextW` / `WriteFile` |
| 4.6 | 关闭窗口时保存 last_file 到配置 + 保存内容 | `Storage_WriteSetting(L"last_file", ...)` |
| 4.7 | 标题栏显示当前文件名（去掉路径） | `PathFindFileNameW` |
| 4.8 | **里程碑**：关掉重开，内容还在；恢复上次编辑的文件 |

**启动流程**：
```
Storage_Init()
    ↓
Storage_ReadSetting("last_file", buf)  → 上次关了哪个文件？
    ↓
├── 有记录 → 打开该文件
└── 无记录 → 创建默认 note.md
    ↓
读取内容 → SetWindowTextW 到编辑器
    ↓
标题栏显示文件名
```

**关闭流程**：
```
WM_CLOSE
    ↓
获取编辑器内容
    ↓
WriteFile 写入 .md
    ↓
Storage_WriteSetting("last_file", 当前文件名)
    ↓
DestroyWindow
```

### 阶段 2（续）：Markdown 实时渲染 + 预览 (Day 13-17) ⏳ 排期中

**目标**：两种模式——编辑模式实时渲染 Markdown 格式，预览模式显示纯渲染效果。

#### 核心架构

```
两个 Rich Edit 控件 + 双击标题栏切换：

编辑模式（默认）                                   预览模式
┌──────────────────────────┐       ┌──────────────────────────┐
│ # 购物清单                │       │ 购物清单（大字加粗）       │
│                          │       │                          │
│ - [ ] **买牛奶**         │  ←→   │ ☐ 买牛奶（加粗）          │
│ - [x] 买鸡蛋 @today      │       │ ☑ 买鸡蛋                 │
│                          │       │                          │
│ @today → 蓝色标签         │       │ today → 蓝色标签          │
│ #购物 → 灰色标签          │       │ #购物 → 灰色标签          │
└──────────────────────────┘       └──────────────────────────┘
   编辑控件（可写）                    预览控件（只读）
   源码可见，格式已生效                 语法标记隐藏
```
