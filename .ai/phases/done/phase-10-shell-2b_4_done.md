# Shell-2b_4 — resize 后布局回归验证（后续）

## 阶段摘要

在缩放实现后，回归验证 resize 过程中和 resize 后标题栏、边框、正文区的布局正确性。

## 阶段目标

1. 验证四边缩放后标题栏高度不变、宽度跟随
2. 验证四角缩放后标题栏和正文区同步缩放
3. 验证快速连续缩放不出现白块或错位

## 为什么要做这个

1. 四边缩放（2b_2）和四角缩放（2b_3）的代码修改只验证了编译和基本功能，没有验证 resize 过程中 UI 布局的稳定性
2. 布局问题可能只在特定的缩放序列或缩放速度下才暴露，需要手动回归
3. 本阶段不修改代码，仅做验证——保证 2b 系列完成后 UI 表现正确

## 阶段产出

1. 确认 resize 后标题栏区域正确
2. 确认 resize 后边框正确
3. 确认 resize 后正文区无错位

## 本阶段范围

不修改代码，仅做手动回归验证。

## 本阶段不做

1. 不修改任何代码
2. 不新增功能

## 技术路线

打开程序，执行以下操作序列并观察：

1. 从左/右/上/下四边拖拽缩放窗口，每次缩放后观察标题栏高度和颜色是否正确
2. 从四角拖拽缩放窗口，观察标题栏和正文区是否同步缩放
3. 快速连续缩放（反复拖动边缘 3-4 次），观察是否出现白块或 UI 错位
4. 缩放后点击菜单按钮弹出菜单，确认功能正常

## 主链路

```text
手动操作
-> 拖拽边缘缩放
-> 观察标题栏/边框/正文区 是否正确
-> 拖拽四角缩放
-> 观察同步性
-> 快速缩放
-> 观察稳定性
```

## 完成标准

1. 四边缩放后标题栏高度不变、宽度跟随，颜色正确
2. 四角缩放后标题栏和正文区同步缩放
3. 快速连续缩放不出现白块或布局错位
4. 缩放后菜单按钮点击仍可弹出菜单
