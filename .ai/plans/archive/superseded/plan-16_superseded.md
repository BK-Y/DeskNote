# Phase 15 - 补高级编辑（后续）

## 阶段摘要

把系统从“单光标编辑”推进到“支持高级编辑模型”。

## 阶段目标

1. 支持多光标
2. 收紧多位置编辑的一致性
3. 为更复杂命令体系建立基础

## 本阶段产出

1. 用户可同时持有多个编辑点
2. 多点插入与删除具有一致行为
3. 高级编辑状态与基础编辑状态边界清楚

## 本阶段范围

1. 多光标状态
2. 多光标下的插入删除
3. 多光标下的渲染与可视反馈

## 本阶段不做

1. 协同编辑
2. 多用户冲突解决
3. 复杂宏系统

## 分层归属

- `editor/`：持有多光标状态与命令规则
- `ui/`：组织多光标可视语义
- `render/`：绘制多个光标 / 选择
- `app/`：流程编排
- `platform/`：只提供输入事件

## 文件落点

### 本阶段预计修改文件

- `src/editor/*`
- `src/ui/*`
- `src/render/*`
- `src/app/*`
- `src/platform/win32/window.c`

### 本阶段原则上不应修改

- `src/storage/note_store.c`
- `src/storage/state_store.c`

## 主链路

```text
输入事件
-> platform
-> app
-> editor 更新多光标状态
-> ui / render 绘制
```

## 完成标准

1. 多光标基础编辑可用
2. 多光标不破坏既有分层
3. 多光标下的可视反馈清楚
