# phase-10-shell-5a_repair-5-a-2 — 从托盘还原时注册 AppBar

目标

- 在用户从系统托盘恢复应用（App_RestoreFromTray）时，重复启动时的注册逻辑，保证托盘还原路径能将窗口恢复到 edge_reserved 状态。

范围（单一小改动）

- 在 `src/app/app.c` 的 `App_RestoreFromTray` 中调用 `App_TryRegisterAppBarFromState()`（由 repair-5-a-1 创建），不再复制复杂逻辑。

实现步骤

1. 确认 repair-5-a-1 中已添加 `App_TryRegisterAppBarFromState(HWND hwnd)` 并导出为 static helper。
2. 在 `App_RestoreFromTray` 的合适位置（在确保窗口句柄和可见性处理之后）调用该 helper。
3. 记录日志（前缀 `[s5a-r5a2]`）以便追踪：记录调用时间、state 值以及注册结果。
4. 错误处理：与 repair-5-a-1 保持一致，失败仅记录，不更改 state。
