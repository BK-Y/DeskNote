# Phase 14 - 补 Markdown 交互（后续）

## 阶段摘要

把系统从“纯文本交互”推进到“理解 Markdown 语义的交互”。

## 阶段目标

1. 支持 Markdown 富文本命中
2. 让点击行为开始区分文本语义与显示语义
3. 为后续样式化与语义交互建立边界

## 本阶段产出

1. 链接、勾选框等语义区域可独立命中
2. 命中不再只是字符位置
3. Markdown 显示与交互链开始联通

## 本阶段范围

1. Markdown 语义命中
2. 语义区域到编辑行为的映射
3. 富文本显示下的最小交互

## 本阶段不做

1. 完整富文本编辑器
2. 多光标
3. 插件化语义系统

## 分层归属

- `core/`：提供 Markdown 语义数据
- `editor/`：收束语义操作后的编辑状态
- `ui/`：组织可交互语义区域
- `render/`：提供语义区域测量与命中
- `app/`：串起交互流程

## 文件落点

### 本阶段预计修改文件

- `src/core/*`
- `src/editor/*`
- `src/ui/*`
- `src/render/*`
- `src/app/*`

### 本阶段原则上不应修改

- `src/storage/note_store.c`
- `src/storage/state_store.c`

## 主链路

```text
语义点击
-> platform
-> app
-> ui 组织语义命中
-> render / core 提供语义区域
-> editor 更新状态
```

## 完成标准

1. Markdown 语义区域可命中
2. 语义交互不打穿分层
3. 纯文本命中与语义命中边界清楚
