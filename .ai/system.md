# AI Working Rules

Before making changes:

1. Read all files in `.ai/`
2. Update `plans.md`
3. Explain architectural impact
4. Record important decisions in `decisions.md`

Rules:

- Never make large changes without updating the plan
- Keep reasoning concise but persistent
- Maintain long-term project consistency
- Prefer editing existing files over generating isolated snippets
- Think step-by-step before coding
